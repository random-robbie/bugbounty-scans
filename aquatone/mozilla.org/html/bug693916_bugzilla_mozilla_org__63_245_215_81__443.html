Created attachment 566442 [details]
Second window in restored state (black title bar)

User Agent: Mozilla/5.0 (Windows NT 6.1; rv:7.0.1) Gecko/20100101 Firefox/7.0.1
Build ID: 20110928134238

Steps to reproduce:

Preconditions:
	- Dual-monitor setup was in "extended display" mode
	- Windows theme was initially set to "Aero"

Exact steps I took:
	1. Launched Firefox.
	2. Maximised the window.
	3. Opened 4 new tabs and directed each tab to a separate YouTube video.
	4. Opened a new FireFox window.
	5. Moved new window to second display and maximised it.
	6. Changed the Windows 7 theme to "Windows Classic".
	7. At this point, visual quirks began to appear such as a strange "edge" around the window. I restored and then maximised the windows.
	8. Changed the Windows 7 theme back to "Aero".
	9. Restored the second Firefox window by dragging the title bar away from the top of the screen.

Description:
	I believe this UI bug is caused by changing the Windows theme while Firefox is running.
	It may only occur when the Firefox windows are in one of the few window states: minimised, restored, maximised.


Actual results:

- The title bar and window controls became black.
- If I attempted to maximise the window again, the title bar would become completely transparent.
- I also wasn't able to close the window by clicking the "x" button (it wasn't there).


Expected results:

- The window theme should have correctly transitioned to the matching Windows 7 theme
